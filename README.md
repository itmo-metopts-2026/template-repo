### Принцип: всё в одном месте

Вся работа ведётся **внутри папки `labs/`**:

- **Реализация** алгоритма — в файле `model.py` рядом с notebook.
- **Исследование** — в `research.ipynb` рядом с `model.py`.
- **Отчёт** — в `report.md` рядом с notebook.
- **Общие утилиты** — в `labs/utils/` (доступны из любой лабы).

## Установка

### Предварительные требования

- Python 3.11+
- [Poetry](https://python-poetry.org/docs/#installation)

### Установка зависимостей

```bash
# Клонирование репозитория
git clone <url-репозитория>
cd <название репозитория>
# Установка зависимостей через Poetry
make install
```

Команда `make install` выполнит `poetry install`, создаст виртуальное окружение
и установит все зависимости с точными версиями из `poetry.lock`.

## Правила оформления лабораторных работ

### С чего начать

1. Откройте [`labs/example/`](labs/example/) — это образцовый пример.
2. Изучите, как устроены `model.py`, `research.ipynb` и `report.md`.
3. Перейдите в папку своей лабораторной (например, `labs/lab1/`).
4. Работайте по шаблону.

### Структура каждой лабораторной

```
labs/labN/
├── README.md          ← Краткое описание и инструкция
├── model.py           ← Реализация алгоритма
├── research.ipynb     ← Jupyter Notebook с экспериментами
├── report.md          ← Отчёт (без кода!)
└── results/
    └── figures/       ← Сохранённые графики
```

### Требования к notebook

1. **Запрещено** писать реализацию алгоритмов в notebook.
2. Notebook импортирует из `model.py`, лежащего рядом.
3. В начале notebook обязательна фиксация `seed`.
4. Все графики сохраняются в `results/figures/`.

### Требования к отчёту

1. Формат — Markdown/PDF.
2. **Код в отчёте запрещён**.
3. Вместо кода — ссылки на файлы реализации (model.py, см. пример лабораторной в labs/example):
```python
from model import *
model.LinearRegression()
```
4. Обязательные разделы: цель, постановка задачи, теория, описание реализации,
   эксперименты, результаты, выводы, воспроизводимость.

### Как сдать лабораторную работу

1. **Проверьте, что всё на месте**  
   В папке лабораторной должны быть: `model.py`, `research.ipynb`, `report.md`,  
   графики в `results/figures/`. Отчёт заполнен, в notebook нет TODO.

2. **Проверьте код** (из корня репозитория):
   ```bash
   make lint    # линтер и проверка форматирования
   make format  # при необходимости автоформатирование
   ```

3. **Закоммитьте и запушьте в Git**:
   ```bash
   git checkout labN 
   git add .
   git commit -m "Лабораторная работа N: краткое описание"
   git push origin labN
   ```

   Затем создайте Pull Request на GitHub.

## Воспроизводимость исследований

Воспроизводимость — ключевое требование научных исследований.
В данном репозитории она обеспечивается следующими мерами:

| Мера | Реализация |
|------|-----------|
| Фиксация случайности | `set_global_seed(42)` в начале каждого notebook |
| Фиксация зависимостей | `poetry.lock` с точными версиями всех пакетов |
| Единая среда | Виртуальное окружение Poetry |
| Документирование | Отчёты и notebook с описанием гиперпараметров |

### Как воспроизвести результаты

1. Установите зависимости: `make install`
2. Запустите JupyterLab: `make jupyter`
3. Откройте notebook нужной лабораторной
4. Выполните все ячейки последовательно: **Kernel → Restart & Run All**

## Разрешённые библиотеки

Набор зависимостей намеренно **минимальный** — суть курса в реализации
алгоритмов оптимизации «с нуля», а не в применении готовых инструментов.

### Для лабораторных работ

| Библиотека | Версия | Назначение | Что можно использовать |
|------------|--------|-----------|----------------------|
| **numpy** | ^1.26.4 | Вычисления | Операции с массивами, матричное умножение, генерация случайных чисел |
| **matplotlib** | ^3.9.2 | Визуализация | Построение графиков, сохранение фигур |
| **jupyterlab** | ^4.2.5 | Среда | Запуск и работа с notebook |


### Что запрещено

Использование готовых реализаций алгоритмов оптимизации и машинного обучения
**запрещено**. В частности, нельзя использовать:

- `sklearn` (scikit-learn)
- `scipy.optimize`
- `torch.optim` / `tensorflow`
- любые другие библиотеки с готовыми оптимизаторами

Цель каждой лабораторной — реализовать алгоритм **самостоятельно**, используя
только `numpy` для матричных операций.

### Добавление новых зависимостей

Если для конкретной лабораторной требуется дополнительная библиотека
(например, `scipy` для сравнения с эталонным решением или `pandas`
для загрузки данных), её можно добавить командой:

```bash
poetry add <имя-пакета>
```

Версия будет автоматически зафиксирована в `poetry.lock`.
